# TODO 2026-06-09：麻将 Agent 下一阶段改进计划

## 目标

下一阶段的核心目标不是单纯提高 PPO 训练 reward，而是建立一个能够证明模型真实进步的训练闭环：

1. 训练环境和 Botzone 部署遵守一致的合法动作与计分规则。
2. 使用固定牌墙和足够样本，对 BC、PPO checkpoint 做可重复的复式评测。
3. PPO 从当前的“单个学习者对启发式与随机策略”升级为带历史模型的混合自博弈。
4. 在训练闭环可信后，引入防守、番型潜力、辅助任务和 Oracle Critic。

最终模型选择必须以实战复式得分为依据，不能只看 BC 准确率、PPO reward 或单次胜率。

## 当前基线与已知问题

### 当前训练方式

- BC 模型学习人类牌谱中的动作分布。
- PPO 从 BC checkpoint 初始化。
- 每局只有一个轮换座位使用正在训练的模型。
- 其他三个座位由启发式策略和随机策略组成。
- 当前不是真正的四模型自博弈，也没有历史 checkpoint 对手池。

### 当前 Reward

终局奖励：

```text
terminal_reward = tanh(player_score / 64)
```

势能塑形：

```text
Phi(s) = -shanten + 0.1 * remaining_useful_tiles
shaping_t = potential_coef * (gamma * Phi(s_{t+1}) - Phi(s_t))
```

默认 `potential_coef=0.02`，终局势能回落至零。PPO 还使用 entropy bonus、value loss、BC-reference KL 正则和 target-KL early stopping。

### 当前主要风险

- `shanten + useful tiles` 主要衡量普通牌效率，不能判断牌型是否能达到国标麻将 8 番。
- Reward 没有显式表达放铳风险、预期损失、流局听牌和番数潜力。
- 随机与启发式对手占比过高，PPO 可能只学会针对弱对手。
- 本地环境和 Botzone 完整规则仍存在差异，可能产生错误 reward 和错误策略。
- 当前 Critic 与 Actor 使用相同的部分可观测信息，价值估计方差较大。
- 训练 reward 上升不等于相对 BC 的复式得分提高。

## P1.5：实现 Oracle Critic / Asymmetric Actor-Critic

### 设计原则

训练时：

```text
Actor:  public_observation -> action probabilities
Critic: privileged_full_state -> state value
```

部署时：

```text
只导出和使用 Actor
```

Actor 永远不能读取其他玩家手牌、实际剩余牌墙或未来摸牌顺序。Oracle Critic 的目标是降低价值估计与 advantage 的方差，不是让部署模型作弊。

### TODO 1.5.1：定义特权状态

- [ ] 为环境增加只供训练使用的 `full_state()` 或 privileged observation。
- [ ] 包含四家手牌计数、公开副露、弃牌、事件和剩余牌种计数。
- [ ] 初版不提供精确未来牌墙顺序，避免 Critic 过拟合模拟器。
- [ ] 明确区分 public observation 和 privileged state 的类型及数据流。
- [ ] 添加测试，保证 Actor 输入中不存在特权信息。

### TODO 1.5.2：拆分 Actor 与 Critic 编码器

- [ ] Actor 保持当前公开信息编码器。
- [ ] Critic 使用独立 privileged encoder，或使用公共 trunk 加 private critic adapter。
- [ ] Rollout 同时保存 Actor observation 和 Critic privileged state。
- [ ] PPO 更新时分别向 Actor 与 Critic 提供对应输入。
- [ ] 可选：保留 public-only value head，用于对照与蒸馏。

风险控制：

- Critic 过强可能造成 Actor/Critic 信息差过大和不稳定更新。
- 模拟器规则不准确时，Oracle Critic 会更确信地学习错误价值。
- 必须监控 explained variance、value loss、advantage 分布和 target KL。

验收标准：

- 在相同牌墙、Reward 和对手池下，Oracle Critic 相比公共 Critic 提高训练稳定性或最终复式得分。
- Actor 导出包不包含特权状态依赖。
- Oracle Critic 没有引入新的非法动作或部署依赖。

## P2：推理搜索、规模化训练与进一步研究

### TODO 2.1：实现轻量 Determinization Rollout / ISMCTS

标准 MCTS 不适合直接用于麻将，因为存在隐藏手牌、随机摸牌、四玩家博弈和较长决策链。
搜索初版采用 determinization：根据公开信息和 belief model 采样若干可能的对手手牌与
剩余牌墙，再对少量候选动作进行有限深度 rollout。

- [ ] 从严格合法动作中选取策略网络 Top-K 候选，初始建议 `K=3-5`。
- [ ] 根据公开牌、不可见牌约束和 belief 分布采样隐藏状态。
- [ ] 保证每个采样状态满足牌数量守恒、公开信息一致和对手暗手牌数量约束。
- [ ] 使用 BC/PPO policy 作为搜索先验，使用 value model 或启发式函数评估叶节点。
- [ ] 对手 rollout 使用快速策略网络或启发式策略，避免搜索成本失控。
- [ ] 聚合不同 determinization 的平均得分与方差，加入风险敏感选择。
- [ ] 搜索结果输出前仍必须通过严格合法动作校验。
- [ ] 增加固定模拟次数和固定时间预算两种模式，记录超时与降级次数。
- [ ] 超时或搜索异常时立即回退原始策略网络动作。

建议初版评分：

```text
search_score(action) =
    mean_rollout_score(action)
    - risk_coef * rollout_score_std(action)
    + prior_coef * policy_log_probability(action)
```

搜索优先用于高影响状态：

- 是否 `CHI/PENG/GANG/BUGANG`。
- 多个听牌或番型路线之间的选择。
- 终盘进攻、防守与危险牌选择。
- 高番潜力与快速和牌之间的权衡。

普通早巡弃牌默认直接使用策略网络，避免为低影响决策支付过高搜索成本。

验收标准：

- Search 相比无 Search 在相同固定牌墙评测中提升复式得分或显著降低放铳率。
- 搜索不增加非法动作，超时与异常均能安全回退。
- 推理延迟满足 Botzone 限制，并报告平均、P95 和最大延迟。
- 比较无 Search、启发式重排序、determinization rollout 和 ISMCTS，证明复杂度确有收益。

### TODO 2.2：经验规则重排序基线

在实现 ISMCTS 前，先建立计算成本较低的经验规则重排序基线：

```text
final_score(action) =
    model_logit(action)
    + lambda_efficiency * tile_efficiency(action)
    + lambda_fan * fan_potential(action)
    - lambda_risk * deal_in_risk(action)
```

- [ ] 所有规则只对严格合法动作重排序，不绕过动作校验。
- [ ] 规则权重保持较小，并与纯模型策略做固定牌墙消融。
- [ ] 记录规则实际改变模型动作的频率和改变后的收益。
- [ ] 若规则重排序已经达到相近收益，暂缓引入高成本搜索。

### TODO 2.3：规模化训练

- [ ] 增加并行环境，提高每次 PPO update 的有效采样量。
- [ ] 实现更完整的四模型自博弈与动态历史对手池。

- [ ] 建立长期 Elo/复式积分榜，持续检测新模型是否退化。

- 平均得分与 95% 置信区间。

- 平均番数和番数分布。
